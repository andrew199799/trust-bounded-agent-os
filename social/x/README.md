# X Content Workflow

This directory keeps lightweight X content assets for public communication around TBAO and related personal-builder themes.

It is not an X API integration and does not post, edit, scrape, automate, or manage the X account.

## Purpose

Use this directory to keep a small, reviewable record of:

- draft posts;
- published posts or profile updates;
- lightweight metrics snapshots;
- reply notes and follow-up ideas.

## Suggested flow

```text
topic idea
  -> draft
  -> human publishes manually on X
  -> published record
  -> metrics snapshot
  -> reply / follow-up notes
  -> next iteration
```

## Directory map

- `drafts/`: reusable post drafts before manual publishing.
- `published/`: records of manually published posts, threads, or profile updates.
- `metrics/`: lightweight snapshots such as views, likes, reposts, replies, bookmarks, or qualitative observations.
- `reply-notes/`: possible replies, follow-up angles, or audience feedback notes.

## Boundary

- No X API keys.
- No automated posting.
- No scraping.
- No account automation.
- No credentials.
- No private messages or personal data.
- Manual publishing remains the only publishing path.

AI may help draft, organize, and review content. The human decides what is published.
