# Friction Log

## Purpose

Track real collaboration friction that should be reviewed during phase review or process tuning.

This log is for inefficiencies, context bloat, boundary leaks, review gaps, validation gaps, and process defects. It is not a progress journal.

## Rules

- Append only when a real friction point is observed.
- Do not record normal progress.
- Keep each entry short.
- Do not include secrets, tokens, personal data, or irrelevant history.
- Do not read this file by default during implementation tasks.
- Use it only for phase review, process tuning, or explicit human request.
