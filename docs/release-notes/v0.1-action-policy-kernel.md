# v0.1 Action Policy Kernel Release Note

## Release Identity

Local-only, mock-only Action Policy Kernel v0.1.

## What Changed

- Added `src/tbao/` local mock kernel.
- Added `ActionRequest`, `PolicyDecision`, `ApprovalTicket`, and `ActionLedgerRecord` models.
- Added `classify_risk`.
- Added `decide_policy`.
- Added `action_hash` and approval ticket validation.
- Added in-memory `ActionLedger`.
- Added L0-L4 examples.
- Added pytest coverage.

## Safety Boundary

```text
local-only
mock-only
non-executing
no credentials
no funds
no external side effects
no live API
no scheduler
no worker
no production runtime
```

## What This Release Proves

- Action risk can be classified.
- L3 can require normal human approval or allowlist.
- L4 is denied by default unless strong approval exists.
- Malformed actions are treated as L4/DENY.
- Ledger records decision paths while keeping `executed=false`.

## What This Release Does Not Prove

- No runtime enforcement.
- No external integration.
- No real-world action safety.
- No production readiness.
- No full Agent OS.
